## 1. 产品定位与核心策略

### 1.1 核心逻辑：高频内容驱动的社交游戏

- **对标产品**：《蛋仔派对》《光遇》。核心在于通过高频内容更新（每月新角色 / 新皮肤 / 新地图）维持用户社交话题与新鲜感，而非依赖单局玩法的深度与耐玩性。
- **产能与规模约束**：团队约 100 人，无法与成熟竞技团队竞争，必须**放弃复杂平衡性维护**，转向"年货式"快速迭代的内容生产模式。
- **反面案例**：《哈利波特：魔法觉醒》——玩法过重导致内容更新缓慢，无法满足低龄用户对新内容的渴求，最终用户流失。

### 1.2 战斗与机制简化

| 决策 | 否决 | 采用 | 理由 |
| --- | --- | --- | --- |
| 战斗底层 | 重度元素反应（《原神》《遥星》） | 轻度物理破坏 + 枪械射击 | 设计过重、不利快速迭代；射击直观易上手、表现力强 |
| 单局深度 | 追求长期耐玩的硬核竞技 | 前 10 小时顺畅体验 | 单局服务于社交场，而非作为终点 |

### 1.3 战斗设计的三条铁律（贯穿全文）

1. **可迭代优先**：任何系统若阻碍"每月一更"的内容产出，一律简化或砍掉。
2. **表现力优先**：技能与破坏必须"好看、好截图、好传播"——社交分享是第一 KPI。
3. **自闭环优先**：英雄/玩法默认按单排体验设计，组队是加分项而非必需项。

### 1.4 单局服务于社交场

```
单局战斗 ──产出──▶ 资源 / 高光时刻 / 拼装单位
                         │
                         ▼
            赛后游乐场（社交展示） ──▶ UGC 创作（地图 / 皮肤 / 拼装）
                         │
                         ▼
                  社交传播 ──▶ 拉新 / 留存
```

## 2. 战斗框架与核心手感

### 2.1 视角与基础操作

| 项 | 方案 | 备注 |
| --- | --- | --- |
| 视角 | 第三人称越肩 / 俯视斜 45°（待 Demo 验证二选一） | 优先利于展示微缩场景与破坏 |
| 移动 | 8 向移动 + 冲刺 | 移动中可射击 |
| 射击 | 自动瞄准辅助 + 手动微调 | 低龄友好，降低上手门槛 |
| 平台 | 移动端优先，兼容 PC | 虚拟摇杆 + 自动开火可选 |

### 2.2 射击手感参数框架（首版基线，待手感打磨）

| 参数 | 基线值 | 说明 |
| --- | --- | --- |
| 基础射速 | 6 发/秒 | 主武器平A |
| 单发伤害 | 10 | 以"对存钱罐 100 HP 需约 16 发"为锚 |
| 弹道 | 即时命中（Hitscan）+ 可视曳光 | 低延迟手感，曳光服务于表现 |
| 后坐力 | 轻微（屏幕 ±2px 抖动） | 不影响低龄玩家命中 |
| 命中反馈 | 准星收缩 + 受击数字 + 音效 + 轻顿帧 | "打到了"必须强反馈 |
| 暴击 | 弱化或取消 | 降低数值理解成本 |

### 2.3 "Juicy"反馈清单（手感打磨验收项）

- [ ] 开火：枪口火光 + 弹壳抛出 + 屏幕轻微推拉
- [ ] 命中：曳光 → 火花 → 飘字 → 命中音
- [ ] 击杀/破坏：顿帧 0.05~0.2s + 镜头微震 + 高光音
- [ ] 拾取：Q弹跳字 + 递增音阶
- [ ] 受击：屏幕边缘红晕 + 方向指示

## 3. 英雄设计框架

### 3.1 标准化英雄模板

> 技能结构：**1 把枪（平A）+ 2 个主动技能**

| 槽位 | 类型 | 设计约束 | 资源关联 |
| --- | --- | --- | --- |
| 平A | 1 把枪 | 构成手感基准与节奏锚点 | 命中可产出/打破破坏物 |
| 技能一（通用） | 位移 / 辅助 / 进攻 三选一 | 保障机动与自保，奠定自闭环 | 无强制 |
| 技能二（破坏） | 与体素资源挂钩 | 把破坏系统嵌入英雄玩法 | **强制**与体素/破坏联动 |

### 3.2 自闭环设计原则

- 英雄必须具备**自保 + 输出 + 一定续航**的闭环，单排能打完整局。
- 避免"必须队友配合才生效"的强协同技能（可作赛季羁绊加分，见 3.4）。
- 技能需具备**高画面表现力**，便于截图与短视频传播。

### 3.3 参考案例拆解 → 落地英雄示例

**Dota2 拆解**

| 原型 | 机制 | 拆解/落地建议 |
| --- | --- | --- |
| 孙悟空 | 七十二变、爬树、分身传送 | 拆成多个独立英雄：变形位移型 / 高台狙击型 / 分身骚扰型 |
| 土猫 | 滚石、拉扯、变石 | "滚石"做成破坏型大招（撞碎一排破坏物并吸体素） |
| 小小 | 投掷、拔树 | "拔树/投掷"= 拾取场景破坏物当投掷物，呼应物理破坏 |

**漫威争锋 参考**

| 原型 | 机制 | 落地建议 |
| --- | --- | --- |
| 奇异博士 | 传送门陷阱 | 关卡交互"传送门"可由英雄技能临时布置 |
| 杰苏 | 漩涡吸吐 | 吸聚体素/敌人，强化收集与控场表现 |
| —— | 场景高度差坠落伤害 | 配合平面化关卡的高台设计 |
| —— | 赛季性羁绊（绿巨人投金刚狼） | 见 3.4 |

### 3.4 英雄设计模板示例（供量产参考）

```
英雄：罐头骑士（示例）
- 平A：散射霰弹枪（近距高伤，远距衰减）
- 技能一（通用·位移）：盾牌冲锋，位移 5m 并格挡前方一次伤害
- 技能二（破坏）：罐盖飞旋——投出罐盖，路径上破坏物碎裂并自动吸附体素
- 自闭环验证：位移自保 + 破坏续航 + 平A输出，可单排
- 社交点：罐盖飞旋的"一路炸碎+金币喷涌"高光镜头
```

### 3.5 赛季羁绊强化（轻度、非必需）

- 每赛季提供少量"羁绊增益"：特定英雄组合触发表演性强化技（如"投掷队友"）。
- 定位为**内容点 + 社交话题**，不进入硬核平衡，单排玩家不受影响。

## 4. 物理破坏系统

### 4.1 资源类型简化 —— 通用资源方案

| 方案 | 描述 | 取舍 |
| --- | --- | --- |
| **采用：通用资源** | 所有可破坏物产出同一种"体素/能量"，仅以**特效 + 材质**区分可破坏性 | 避免美术资源爆炸，利于快速迭代 |
| 不采用：严格材料分类 | 区分木材/石材/金属等 | 资源与配置成本爆炸，违背可迭代铁律 |

> 注：视觉上可保留"陶瓷=冰蓝体素、金币=金色体素"等**颜色区分**用于表现，但底层结算仍为通用资源。

### 4.2 整体破坏 vs 局部破坏（逻辑博弈）

| 维度 | 整体破坏（首版采用） | 局部破坏（后续演进） |
| --- | --- | --- |
| 逻辑复杂度 | 低（单一血条→整体替换碎裂） | 高 |
| 战术深度 | 较浅 | 深（打掉一角改变地形/掩体） |
| 待解问题 | —— | 资源归属、地形改变、多模型配置 |
| 结论 | **Demo 阶段用整体破坏控成本** | 验证留存后再做局部破坏 |

### 4.3 破坏物状态机

```
[完好] ──受击──▶ [受损·阶段裂纹] ──HP≤0──▶ [爆裂演出] ──▶ [体素化] ──▶ [吸附] ──▶ [销毁]
   │                  │
   └── 持续展示裂纹贴图随 HP 加密 ┘
```

| 状态 | 进入条件 | 表现 | 可交互 |
| --- | --- | --- | --- |
| 完好 | 默认 | 原始模型 | 可被击中 |
| 受损 | HP < 100% | 裂纹贴图 + 受击闪烁/形变 | 可被击中 |
| 爆裂 | HP ≤ 0 | 顿帧+子弹时间+物理破碎+喷涌 | 否 |
| 体素化 | 散落停留结束 | 模型解构为发光体素 | 否 |
| 吸附 | 体素悬浮 0.5s 后 | 蜂群曲线飞向玩家 | 否 |

### 4.4 受击阶段（以巨型存钱罐为标杆案例）

**设定**：表面光滑圆润、内部装满高价值资源；掉落高级货币/核心能源。设计目的——突出射击陶瓷材质的清脆反馈，以及爆裂时"财富喷涌"的极致解压感。

| HP 阈值 | 裂纹密度 | 透光 | 音效叠加 |
| --- | --- | --- | --- |
| 100% | 无 | 无 | "叮、叮"清脆瓷声 |
| 75% | 初现网状裂纹 | 缝隙微透金光 | 叮 + 偶发"咔" |
| 50% | 裂纹加密 | 明显透金光 | 叮 + 持续"咔咔" |
| 25% | 密集裂纹·濒裂 | 强透光 | "咔咔"开裂为主 |
| 0% | —— | —— | 触发爆裂演出 |

- 材质交互：子弹命中产生**跳弹火花** + **白色陶瓷粉末**飞溅。
- 受击闪烁：每次命中产生轻微**受击形变** + **白光闪烁**。

### 4.5 终结破坏演出（爆裂阶段）

| 阶段 | 时序 | 表现 |
| --- | --- | --- |
| 顿帧 | 命中瞬间 0.2s | 全局顿帧放大张力 |
| 子弹时间 | 顿帧后极短 | 慢放爆裂瞬间 |
| 物理破碎 | 即时 | 陶瓷壳解体为数十块大小不一的物理碎片 |
| 内部喷涌 | 即时 | 金币/纽扣如喷泉爆射 + 环形气浪（冲击波特效） |
| 听觉 | 爆裂→落地 | "哗啦——砰！"陶瓷碎裂 → 密集"叮叮当当"金币落地 |

### 4.6 三阶段收集演出（核心解压爽点）

> 设计意图：先**物理散落**带来视觉满足，后**体素化吸附**带来魔法感的独特节奏。

| 阶段 | 时长 | 表现 | UI/音频 |
| --- | --- | --- | --- |
| ① 物理散落与沉淀 | 1.0~1.5s | 碎片与金币受物理引擎驱动，弹跳翻滚至静止，地面保持实体停留 | 落地碰撞音；给玩家欣赏"爆了一地"的窗口 |
| ② 体素化解构 | ~0.5s | 从边缘解构为发光体素：陶瓷→冰蓝光点、金币→金色光点，伴"嗡嗡"充能并微微悬浮升起（失重感） | 充能音 |
| ③ 自动吸附入体 | 悬浮 0.5s 后触发 | 无需按键，自动锁定玩家；光点如蜂群先外扩再以流体曲线、拖流光尾迹急速飞向胸口/背包 | 撞击爆微粒火花；资源栏Q弹跳字（陶瓷+10 / 金币+500）；街机吃币式连续音、音阶随数量递增推向高潮 |

**收集参数框架**

| 参数 | 基线 | 说明 |
| --- | --- | --- |
| 物理停留 | 1.0~1.5s | 过短无丰收感，过长拖节奏 |
| 体素悬浮 | 0.5s | 失重蓄势 |
| 吸附飞行 | 0.4~0.7s | 蜂群曲线，非直线 |
| 自动吸附半径 | 全屏（无需靠近） | 降低操作负担 |

## 5. 场景交互与关卡设计

### 5.1 交互元素

| 元素 | 作用 | 实现要点 |
| --- | --- | --- |
| 弹簧 | 弹射位移，制造垂直空间博弈 | 触发即施加向上冲量，配合坠落伤害 |
| 传送门 | 快速转场 / 战术绕后 | 成对布置，可固定或英雄技能临时布置 |
| 陷阱 | 环境杀 / 控场点 | 触发型，配合击退把敌人推下高台 |

### 5.2 关卡结构

- 参考《Star Trek》的**平面化关卡结构**，在此基础上融合物理破坏元素。
- 平面化降低设计复杂度，**确保关卡产出能支撑快速迭代**（每月新图）。
- 关卡需预留：破坏物布点、捐赠提交点、高台/坠落区、交互元素位。

### 5.3 关卡可迭代清单（模板化产出）

- [ ] 一张底图 + 可替换主题皮肤（沙漠/糖果/文具桌面等）
- [ ] 标准化破坏物预制体库（通用资源结算）
- [ ] 交互元素预制体（弹簧/门/陷阱）
- [ ] 战略建造点 × N
- [ ] 出生点 / 复活点 / 高光展示位

## 6. 团队战略建造系统（捐赠超级兵）

### 6.1 机制

- 地图内设特定"**捐赠提交点**"，团队成员将收集的材料上交，进度条满后**自动生成战略单位**（公共消耗）。
- **核心特色（所见即所得的拼装感）**：根据提交的**主要材料类型**，生成单位在视觉与能力上差异化表现。

### 6.2 战略单位表

| 类别 | 系别 | 单位 | 能力定位 |
| --- | --- | --- | --- |
| 进攻型（自动推进） | 文具/杂物系 | 纸箱与废品拼装的"文具高达" | 巨型推进单位 |
| 进攻型 | 建材系 | 橡果壳炮塔、发射爆炸松果的战车 | 远程压制 |
| 进攻型 | 混合/特殊系 | 拼布大魔王（毛绒玩具熊/兔） | 高血量肉盾推进 |
| 防御型（阵地保护） | 食物系 | 姜饼人射手 | 火力支援 |
| 防御型 | 其他 | 蘑菇穹顶保护伞 | 范围减伤 / 护盾 |

### 6.3 数值与触发框架（首版）

| 参数 | 基线 | 说明 |
| --- | --- | --- |
| 进度条容量 | 100 单位材料 | 团队共同填充 |
| 主材判定 | 占比最高的材料类型 | 决定生成单位外观/能力 |
| 单位存活 | 限时 / 限血 | 防止滚雪球，服务节奏 |
| 提交点数量 | 1~2 / 局 | 制造团队争夺焦点 |

## 7. 资源系统

| 层级 | 类型 | 来源 | 用途 |
| --- | --- | --- | --- |
| 局内通用资源 | 体素/能量（统一结算） | 破坏物 / 击杀 | 战略建造捐赠、技能充能 |
| 局内高级资源 | 高级货币 / 核心能源 | 高价值破坏物（存钱罐等） | 强力单位/技能 |
| 局外（社交场） | 货币 / 装扮 | 单局结算 | 皮肤/角色/UGC |

> 资源底层统一为"通用资源"，**颜色与特效用于表现差异**，不增加结算复杂度（呼应 4.1）。

## 8. 单局节奏与流程

```
开局(集结/铺垫) ─▶ 资源采集与破坏(解压循环) ─▶ 战略建造博弈(团队焦点)
      ─▶ 高潮对抗(超级兵推进) ─▶ 结算 ─▶ 赛后游乐场(社交)
```

| 阶段 | 目标 | 关键体验 |
| --- | --- | --- |
| 前期 | 上手顺畅 | 射击手感 + 破坏解压 |
| 中期 | 资源争夺 | 捐赠点博弈 |
| 后期 | 一锤定音 | 超级兵推进高光 |
| 赛后 | 社交沉淀 | 游乐场 / UGC |

## 9. 技术实现要点

| 模块 | 要点 | 风险/对策 |
| --- | --- | --- |
| 破坏 | 整体破坏=预制碎裂体替换；体素化=GPU 粒子/Niagara | 局部破坏暂不做，控成本 |
| 收集 | 物理散落用刚体，吸附阶段切为粒子蜂群曲线 | 移动端碎片数量分级（LOD/降配） |
| 通用资源 | 单一资源类型结算，表现层做颜色/特效区分 | 避免材料分类爆炸 |
| 顿帧/子弹时间 | 全局时间缩放 TimeDilation | 多人同步需服务器权威或表现层本地化 |
| 自动吸附 | 全屏锁定，客户端表现 + 服务器结算 | 防作弊：结算以服务器为准 |
| 英雄模板 | 数据驱动技能（配置表 + 通用技能基类） | 支撑"每月新英雄"量产 |

## 附录 C：竞品角色拆解 ·《植物大战僵尸：花园战争 2》僵尸阵营

> 来源：B 站《【花园战争2】全部僵尸变种角色简介》。该作是「第三人称卡通英雄射击 + 皮肤式变种」的标杆，与本项目同品类同视角，**用于校准本项目英雄模板与内容产能模型**。完整拆解见《参考拆解_花园战争2僵尸阵营.md》，此处保留可直接用于本案的结论。

### C.1 对标价值（一句话）

花园战争2 = 一个**已验证的「英雄射击 + 皮肤式变种」量产模型**，替本项目趟过了「有限产能如何持续产新角色」这条路。它用「1 武器 + 3 技能」，本项目砍为「1 枪 + 2 技能」即可落地。

### C.2 按「手感原型」铺角色（而非按外观）

僵尸 7 职业本质是 7 种**手感原型**。本项目首发英雄应先占满 4~6 种**截然不同的手感**，而非做多个都拿突击枪只换皮的角色——手感差异 = 玩家「换角色」的核心动力。

| 手感原型 | 对标职业 | 战术定位 |
| --- | --- | --- |
| 中距全自动 | 步兵 Foot Soldier | 全能 / 入门位 |
| 高射速前排 | 全明星 All-Star | 坦克 / 冲锋（自带掩体+撞人） |
| 近距霰弹+瞬移 | 科学家 Scientist | 治疗 / 近身爆发（奶刺客） |
| 抛物线榴弹+召唤 | 工程师 Engineer | 控场 / 召唤（非枪法位） |
| 双枪+变身机甲 | 小鬼 Imp | 机动骚扰 + 短时强力形态 |
| 双形态狙击 | 海盗船长 Deadbeard | 开镜=狙、腰射=霰弹 |
| 近战+蓄力光束 | 超级英雄 Super Brainz | 近战缠斗 |

### C.3 变种系统 = 本项目月更产能的标准答案

40+ 个变种角色，玩家感知是「40 个新角色」，研发只维护 **7 套技能骨架 + 5 种武器改法 + 外观主题**：

| 武器改法 | 含义 | 美术成本 | 玩法增量 |
| --- | --- | --- | --- |
| 精准 | 远程低扩散、拉远有效距离 | 中 | 中-高 |
| 速射 | 高射速低单发 | 低-中 | 中 |
| 重击 | 单发高伤、慢射速 | 低-中 | 中 |
| 元素 | 平A 附加火/冰/毒/电 | 低-中 | 中 |
| 特形 | 改交战距离/瞄准方式/弹道形态 | 中-高 | 高 |

> **产能结论**：本项目「每月新角色」= 在骨架上排列组合武器改法 + 换主题皮肤；真正从零做技能动画的**全新骨架，一季度只需 1~2 个**，其余靠变种填充月度更新。直接支撑第 12 章 KPI「每月稳定产出 1 英雄 + N 皮肤」。

### C.4 首发英雄手感原型铺排建议（接本案 3.1「1 枪 + 2 技能」模板）

| 原型 | 参考职业 | 技能一（通用·位移） | 技能二（破坏挂钩） | 微缩世界演出点 |
| --- | --- | --- | --- | --- |
| 全自动突击 | 步兵 | 火箭跳上高台 | 火箭弹炸塌存钱罐/蛋糕 | 标准入门位 |
| 重机枪前排 | 全明星 | 冲撞开团 | 撞碎可破坏掩体 | 巨物撞翻微缩家具 |
| 霰弹奶/刺客 | 科学家 | 短瞬移 | 黏弹定向爆破取体素 | 瞬移穿梭微缩房间 |
| 抛射召唤 | 工程师 | 钻机加速 | 榴弹拐弯炸掩体后 | 无人机俯瞰微缩场景 |
| 双形态机甲 | 小鬼 | 灵活走位 | **变身巨型机甲践踏拆楼** | 小人↔巨物反差，强传播点 |
| 近战缠斗 | 超级英雄 | 飞踢突进 | 旋风绞碎周围可破坏物 | 近身撕扯微缩布景 |

### C.5 可直接搬用的 5 条设计法则

1. **每个角色必含一个「位移/保命」技能** → 对应本案 3.2「自闭环」。
2. **每个角色至少一个「高表现力技能」**（为截图/录屏而设计）→ 对应本案铁律「表现力优先」。
3. **武器手感先行，技能服务于手感**（霰弹短腿就给位移，近战没射程就给光束）。
4. **职业克制代替复杂平衡**（远程vs近战/机动vs阵地），而非元素反应数值网 → 对应本案「放弃重度元素反应」。
5. **零血腥、强反馈**：受击/破坏/击杀全部夸张卡通特效 → 直接对应本案 4.x「存钱罐爆裂、产出体素」演出。
